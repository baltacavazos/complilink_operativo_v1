# Auditapatron: insumo para auditoría de simplicidad extrema

## Objetivo

Evaluar si la plataforma es suficientemente intuitiva para cualquier trabajador de a pie en México. La revisión debe detectar todo lo que suene técnico, intimidante, redundante, demasiado largo o poco obvio.

## Landing: primer pantallazo actual

> Tu recibo puede darte una primera lectura clara.
>
> Sube un recibo, CFDI o PDF y te mostramos la señal principal y el siguiente paso.
>
> Empieza con un solo documento y entiende rápido si hay algo que revisar.
>
> Primero ves la lectura y después decides si quieres seguir o guardar tu expediente.

Elementos visibles relevantes:

- Navegación con: Asegurar evidencia, Privacidad, Cómo funciona, Preguntas, Iniciar sesión, Empezar auditoría gratis.
- Tarjeta de ayuda: “Si no sabes con qué empezar”.
- Ejemplo visual de resultado con señal encontrada.
- CTA principal repetido varias veces.

## /auditar: contenido visible actual

> Sube tu documento
>
> Sube una foto, PDF o XML. Primero verás la señal principal y el siguiente paso útil.
>
> Archivo protegido y usado solo para tu auditoría.
>
> Sube tu archivo y mira la lectura inicial.
>
> Primero ves el documento, la señal y el siguiente paso antes de guardarlo.

Elementos visibles relevantes:

- Botones: Cámara y Archivo.
- Texto de apoyo: “Archivo principal”, “Confidencial”, “Protegido”, “Bajo tu control”.
- Texto técnico/operativo aún visible: “Escaneo asistido por IA”, “Nitidez y legibilidad”, “Bordes y orientación”, “Coincidencia con el documento esperado”.
- Mensajes de proceso: “Etapa 1 Preparar”, “Etapa 2 Analizar”, “Etapa 3 Guardar”, “Etapa 4 Revisar”.
- Texto adicional: “bóveda laboral”, “Contexto opcional”, “12 MB PDF · XML · imagen”.

Elementos visibles relevantes:

- Botón principal: “Continuar con mi auditoría”.
- Enlace secundario: “Volver a /auditar”.

## Hipótesis iniciales detectadas manualmente

- La landing ya mejoró, pero todavía puede tener demasiados CTAs y términos algo abstractos para un usuario muy básico.
- En /auditar todavía sobreviven palabras más propias del producto que del usuario común: IA, expediente, bóveda laboral, contexto opcional, etapas, legibilidad.
- El flujo debería sentirse como: “sube tu recibo, mira el resultado, decide si sigues”; cualquier cosa fuera de eso compite con la comprensión inmediata.
- El tono debe sonar mexicano, cercano, laboral y cero técnico.
